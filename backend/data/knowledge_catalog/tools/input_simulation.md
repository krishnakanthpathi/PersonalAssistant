# macOS Input Simulation & Hardware Automation

The **Input Simulation** category provides low-level hardware input simulation capabilities for macOS, enabling automated control over keyboard and mouse devices. These tools generate native system events to interact with applications as if a physical user were present.

### Available Tools

| Tool | Description |
|------|-------------|
| **`keystroke_action`** | Simulates typing text or pressing specific keyboard shortcut keys on macOS using native events. |
| **`key_press`** | Press a single key or shortcut (named key + modifiers) on macOS. |
| **`type_text`** | Type a Unicode string into the active input field. |
| **`mouse_move`** | Move the mouse cursor to global screen coordinates (x, y). |
| **`mouse_click`** | Click at (x, y) with optional button, count, and modifiers. |
| **`mouse_drag`** | Press, drag, and release mouse from (fromX, fromY) to (toX, toY). |
| **`mouse_scroll`** | Send a scroll-wheel event in line or pixel units. |
| **`run_applescript`** | Executes custom raw AppleScript (osascript) for advanced scripting, UI automation, window management, simulating mouse clicks, keystrokes, or any system automation not supported by other tools. |
